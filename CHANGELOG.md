# Changelog

_Maintained by Chorus._

- **2026-06-15T03:04:32.599Z** _(by backend-dev)_ — Implement orchestrator task loop — Implemented the orchestrator task loop so ready tickets are selected by priority while respecting active role and idle-agent availability, persisted as task/run work with project context, role context, guardrails, ticket details, and allocated branch state, then reconciled into ticket/task/agent lifecycle updates as runs finish. The loop now uses persisted active tasks and runs to avoid duplicating work across service restarts, and includes coverage for dispatch, branch allocation, completion reconciliation, and restart dedupe.
- **2026-06-15T02:53:03.514Z** _(by backend-dev)_ — Add Codex CLI backend adapter — Added a Codex CLI backend adapter that runs `codex exec` non-interactively in a specified repo worktree, records run lifecycle data, persists captured stdout/stderr and log payloads to run records, classifies success, failures, quota exhaustion, and cancellations, and exposes cancellable run handles for orchestrator control. Integration coverage uses fake Codex executables for success, failure, quota, and cancellation paths.
- **2026-06-15T02:44:37.672Z** _(by backend-dev)_ — Implement built-in ticket tracking — Implemented a minimal built-in ticketing system with public APIs for creating, editing, assigning, reprioritizing, transitioning, closing, listing, and reading tickets; ticket bodies are now persisted, lifecycle validation covers backlog, ready, in progress, review, merged, blocked, and done, ticket responses expose linked agent runs through task/run relationships, and project resume snapshots now include ticket state.
- **2026-06-15T02:37:06.343Z** _(by backend-dev)_ — Discover project specs on import — Implemented project context discovery on import: cloned repos are scanned for common spec and architecture files, discovered references are persisted on the project, missing required context moves the project into `needs_context`, pasted/uploaded context content can be stored through a public helper, and orchestration startup now blocks until required context exists.
- **2026-06-15T02:29:50.999Z** _(by backend-dev)_ — Create project import flow with gh — Added a gh-backed project import flow that validates submitted GitHub repository URLs, requires `gh auth status` to succeed before cloning, clones via `gh repo clone`, detects the cloned repository’s default branch locally, and persists the repo URL, local path, default branch, integration branch, and import provenance in project metadata with clear typed errors for auth, URL, clone, and branch-detection failures.
- **2026-06-15T02:24:23.537Z** _(by backend-dev)_ — Define core persistence schema — Selected SQLite as the durable local store and added a packaged migration that creates the core Chorus persistence schema for projects, roles, guardrails, agents, tickets, tasks, runs, quota samples, branch state, merges, notifications, and changelog entries. Added a small repository layer with JSON field handling, timestamped create/read/update/list flows, restart-safe persistence tests, README usage notes, and committed the work as `eea24a8 Define core persistence schema`.
